## Content Basics

### Content Types

Simpla has three content types:

| Type | Location | Purpose |
|------|----------|---------|
| **Posts** | `content/posts/` | Blog articles, shown in indexes and feeds |
| **Pages** | `content/pages/` | Static pages (about, contact, legal) |
| **Snippets** | `content/snippets/` | Reusable content blocks for tag pages |

### File Format

All content files are **Markdown** (`.md`) with **YAML frontmatter**:

```markdown
---
title: 'My Article Title'
status: 'published'
created_at: '2025-01-02'
---

Your content goes here in **Markdown**.
```

### File Naming

The filename determines the URL slug (unless you override it with `slug` in frontmatter):

| Filename | Generated URL |
|----------|---------------|
| `about.md` | `/about.html` |
| `my-first-post.md` | `/my-first-post.html` |
| `Hello World.md` | `/hello-world.html` |

> **Tip:** Use lowercase with hyphens for cleaner URLs: `my-first-post.md`

## Creating Static Pages

Pages are stored in `content/pages/` and are ideal for:
- Homepage
- About page
- Contact information
- Legal pages (imprint, privacy policy)

### Basic Page

Create `content/pages/about.md`:

```markdown
---
title: 'About Me'
status: 'published'
created_at: '2025-01-02'
---

# About

Hi, I'm Alex! I'm a web developer passionate about...

## My Work

I specialize in:
- Frontend development
- Static site generators
- Coffee brewing (very important!)

## Contact

Feel free to reach out at hello@example.com
```

### Custom Homepage

To use a page as your homepage, set `slug: 'index'`:

```markdown
---
title: 'Welcome'
slug: 'index'
status: 'published'
created_at: '2025-01-02'
template: 'homepage.phtml'
---

# Welcome to My Site

This is the homepage content.
```

### Page with Custom Template

Use a different template for specific pages:

```markdown
---
title: 'Contact'
status: 'published'
created_at: '2025-01-02'
template: 'contact.phtml'
---

Contact information...
```

Then create `views/[theme]/contact.phtml` with your custom layout.

## Frontmatter Reference

Frontmatter is YAML metadata at the top of your Markdown file, enclosed by `---`.

### Required Fields

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Content title (appears in headings, meta tags) |
| `status` | string | `'published'` or `'draft'` |
| `created_at` | date | Publication date in `YYYY-MM-DD` format |

### Optional Fields (All Content)

| Field | Type | Description |
|-------|------|-------------|
| `slug` | string | Custom URL slug (overrides filename) |
| `author` | string | Author name (overrides config default) |
| `description` | string | Custom meta description for SEO |
| `template` | string | Custom template filename |

### Post-Only Fields

| Field | Type | Description |
|-------|------|-------------|
| `tags` | string | Comma-separated tag names |
| `image` | string | Featured image filename (from `content/img/`) |

### Examples

**Minimal post:**
```yaml
---
title: 'Quick Update'
status: 'published'
created_at: '2025-01-02'
---
```

**Full-featured post:**
```yaml
---
title: 'Advanced Techniques in Modern CSS'
status: 'published'
created_at: '2025-01-02'
author: 'Alex Designer'
tags: css, frontend, tutorial, advanced
image: css-grid-layout.png
description: 'Master CSS Grid, Flexbox, and custom properties with practical examples.'
slug: 'css-advanced-techniques'
---
```

**Frontmatter data types supported:**
- Strings (quoted or unquoted)
- Booleans: `true` / `false`
- Numbers: `42`, `3.14`
- Null: `null` or `~`
- Lists: inline (`[a, b, c]`) or block:

```yaml
---
tags:
  - tag-a
  - tag-b
---
```

**Page with custom slug:**
```yaml
---
title: 'Impressum'
slug: 'imprint'
status: 'published'
created_at: '2025-01-02'
---
```

## Markdown Guide

Simpla uses a built-in Markdown parser that supports common Markdown syntax.

### Text Formatting

```markdown
**bold text**
*italic text*
***bold and italic***
~~strikethrough~~
`inline code`
```

### Headings

```markdown
# Heading 1
## Heading 2
### Heading 3
#### Heading 4
```

### Links

```markdown
[Link text](https://example.com)
[Link with title](https://example.com "Title")

<!-- Internal links -->
[About page](/about.html)
[Another post](/my-other-post.html)
```

### Images

```markdown
![Alt text](/img/photo.jpg)
![Alt text](/img/photo.jpg "Image title")
```

### Lists

```markdown
Unordered list:
- Item one
- Item two
  - Nested item
  - Another nested

Ordered list:
1. First item
2. Second item
3. Third item
```

### Tables

```markdown
| Column | Description |
|:-------|:------------|
| First  | Left aligned |
| Second | Centered     |
```

### Blockquotes

```markdown
> This is a quote.
>
> It can span multiple paragraphs.
```

### Code Blocks

````markdown
Inline: Use `const` for constants.

Fenced code block with syntax highlighting:

```javascript
function greet(name) {
    console.log(`Hello, ${name}!`);
}
```

```python
def greet(name):
    print(f"Hello, {name}!")
```
````

Language identifiers after the opening backticks (e.g., ```` ```javascript ````) are added as `language-...` classes on `<code>` tags for syntax highlighting.

### Horizontal Rule

```markdown
---
```

### HTML in Markdown

You can use raw HTML when needed:

```markdown
Regular markdown paragraph.

<div class="custom-box">
  <p>This is a custom HTML block.</p>
</div>

Back to markdown.
```

### Escaping HTML

Simpla leaves raw HTML untouched by default. If you need to sanitize untrusted content (for example when injecting external sources), instantiate the markdown parser with escaping enabled:

```php
$parser = new \Simpla\Markdown\MarkdownParser(true); // escape HTML
```

## Tags and Categories

Tags help organize your blog posts and create automatic archive pages.

### Adding Tags to Posts

```yaml
---
title: 'Learning JavaScript'
tags: javascript, tutorial, beginner
---
```

- Tags are **comma-separated**
- They're **case-insensitive** (normalized to lowercase in URLs)
- Spaces are converted to hyphens

### Generated Tag Pages

For each unique tag, Simpla creates a page at:

```
/tags/[tag-name].html
```

Example:
- `tags: javascript` â†’ `/tags/javascript.html`
- `tags: static-sites` â†’ `/tags/static-sites.html`

### Linking to Tag Pages

In your templates or content:

```markdown
Check out all my [JavaScript posts](/tags/javascript.html)!
```

### Tag Page Descriptions (Snippets)

You can add a description to tag pages using snippets. See the [Snippets](#snippets) section.

## Snippets

Snippets are reusable content blocks, primarily used for tag page descriptions.

### Creating a Tag Snippet

Create `content/snippets/[tag-name].md`:

```markdown
---
title: 'javascript'
status: 'published'
created_at: '2025-01-02'
---

### About JavaScript

JavaScript is a versatile programming language primarily used for web development.
On this page, you'll find all my posts about JavaScript, from beginner tutorials
to advanced techniques.
```

### How Snippets Work

When generating the `/tags/javascript.html` page, Simpla:

1. Looks for `content/snippets/javascript.md`
2. If found, includes its content at the top of the tag page
3. Lists all posts with that tag below

## Content Workflow Tips

### Local Preview Workflow

1. **Edit content** in your favorite editor
2. **Rebuild** the site:
   ```shell
   docker run -it --rm \
     -v "$PWD/page":/usr/src/simpla/page \
     -v "$PWD/dist":/usr/src/simpla/dist \
     fanatique/simpla:latest
   ```
3. **Preview** in browser (keep `php -S localhost:8080` running in `dist/`)
4. **Repeat**

### Quick Rebuild Script

Create `build.sh` in your project root:

```bash
#!/bin/bash
docker run -it --rm \
  -v "$PWD/page":/usr/src/simpla/page \
  -v "$PWD/dist":/usr/src/simpla/dist \
  fanatique/simpla:latest

echo "âœ… Build complete! View at http://localhost:8080"
```

Make it executable:
```shell
chmod +x build.sh
./build.sh
```
